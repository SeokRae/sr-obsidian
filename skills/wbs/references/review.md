# /wbs review — 품질 점검

## 5가지 점검 규칙

**① 100% Rule**: phase-1.md ~ phase-8.md + 세부 분리 step 파일 전부 존재하는가?

**② 8/80 Rule**: `**기간**: N일` 파싱 → N > 3 이면 위반 (분해 권장)

**③ 산출물 누락**: `**산출물**:` 값이 비어있는 Task

**④ 의존성 순환**: A→B→A 패턴 감지

**⑤ 중복**: 동일 Task명 2회 이상 등장

## 출력 형식

```markdown
## /wbs review 결과

### 품질 점검 리포트

✅/⚠️ **100% Rule**: {결과}
✅/⚠️ **8/80 Rule**: {N건 위반}
  {위반 Task 목록}
✅/⚠️ **산출물 누락**: {N건}
✅/⚠️ **의존성 순환**: {결과}
✅/⚠️ **중복 없음**: {결과}

### 요약
| 항목 | 결과 |
|------|------|
| 총 Task 수 | N개 |
| 총 예상 공수 | N일 |
| 위반 사항 | N건 |

### 수정 제안
{위반 사항별 구체적 수정 방안}

### 다음 단계
위반 사항 수정 후 `wbs-status`를 `export`로 업데이트하세요.
```
