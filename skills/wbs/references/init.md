# /wbs init — 프로젝트 정보 수집

## 수집 항목

| # | 항목 | 질문 | 예시 |
|---|------|------|------|
| 1 | 프로젝트 유형 | `web-fullstack` / `api-backend` / `mobile-app` / `batch-data` 중 무엇인가요? | `api-backend` |
| 2 | 프로젝트명 | 프로젝트 이름은 무엇인가요? | `결제 중계 플랫폼` |
| 3 | 기술 스택 | FE/BE/DB/Infra 기술 스택은? | `Spring Boot 3.2, Oracle 19c, IDC` |
| 4 | 팀 구성 | 역할별 인원은? | `PM 1, BE 3, FE 2, QA 1, DBA 1` |
| 5 | 일정 | 총 기간과 시작일은? | `4개월, 2026-04-01` |
| 6 | 비즈니스 도메인 | 핵심 도메인 목록은? | `결제 승인, 정산, 회원, 가맹점` |
| 7 | 사전 문서 유무 | 요구사항/설계 문서가 있나요? | `요구사항 정의서 있음, 설계 없음` |
| 8 | 세부 분리 대상 Phase | 5개 이상 태스크·외부 협력·개별 산출물 필요한 Phase 있나요? | `인프라: ITSM 13단계` |
| 9 | 외부 어댑터/연동 시스템 | 외부 어댑터 2개 이상이면 목록과 약칭을 알려주세요. | `NicePay(nice), Stripe(stripe)` / `없음` |

항목별로 질문하거나 한 번에 파싱. 불명확한 항목만 추가 질문.

> **항목 8 분리 기준** — 아래 중 2개 이상 해당:  
> 단계 수 5개 이상 / 외부 팀 신청→승인 사이클 / 원문·실제 값 기록 필요 / 단계별 독립 참조 필요

> **항목 9 분리 기준** — 아래 중 하나 해당:  
> 외부 시스템 2개 이상 / 어댑터별 독립 도메인·기능 / 어댑터별 진행률 별도 추적 필요  
> → 분리 시 `dev-{약칭}` 복수 group 사용. 약칭 없으면 소문자 앞 4자.

| 세부 분리 Phase | {없음 \| 인프라(13단계), ...} |
